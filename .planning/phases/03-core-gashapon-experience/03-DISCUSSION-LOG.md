## 动画形式

| Option | Description | Selected |
|--------|-------------|----------|
| 球形胶囊从杯口弹出 | 透明球从奶茶杯顶部弹出→旋转→落下的动画 sequence | |
| 整杯震动 + 胶囊飞出 | 奶茶杯先剧烈晃动，然后胶囊弹射出来 | |
| 杯盖打开 + 胶囊升起 | 杯盖像门一样打开，胶囊从内部缓缓升起 | |

**User's choice:** 用户提出新想法——从吸管吸出来，模拟喝珍珠奶茶的场景
**Notes:** 用户否定了所有预设选项，给出了更贴合奶茶主题的创意：从吸管吸出珍珠。这个想法完美融合了"扭蛋惊喜"和"喝奶茶"两个体验。

## 弹出位置

| Option | Description | Selected |
|--------|-------------|----------|
| 从杯口弹出（推荐） | 胶囊从奶茶杯盖下方升起 | |
| 从杯底弹出 | 胶囊从杯子下方滚落 | |
| 从杯身正面弹出 | 胶囊从杯身正前方弹出 | |
| **从吸管吸出（用户创意）** | **模拟珍珠奶茶喝珍珠的场景** | ✓ |

**User's choice:** 从吸管吸出来，模拟喝珍珠奶茶的珍珠的场景
**Notes:** 这是本次讨论最关键的创意突破。胶囊不再是独立的"玩具"，而是奶茶体验的一部分。扭蛋=喝奶茶。

## 动画轨迹

| Option | Description | Selected |
|--------|-------------|----------|
| 吸管内上升→顶部弹射 | 胶囊先在吸管内加速上升，到达顶部后弹射出来在半空打开 | |
| 吸管内螺旋上升（推荐） | 胶囊像真珍珠一样在吸管内旋转着上升，到顶后弹出打开 | ✓ |
| 吸管底部闪一下→弹出 | 更简洁：吸管底部闪烁一下，胶囊直接弹射出来 | |

**User's choice:** 吸管内螺旋上升
**Notes:** 更符合真实珍珠奶茶的体验，珍珠在吸管内会旋转着上升。

## 云端概率调用时机

| Option | Description | Selected |
|--------|-------------|----------|
| 点击即刻并行（推荐） | 点的同时触发动画+云端调用，动画播完结果也回来了，几乎无等待 | ✓ |
| 等结果再播动画 | 先等云端返回"选中哪颗"，再播动画展示结果，延迟感强但更戏剧化 | |

**User's choice:** 点击即刻并行（推荐）
**Notes:** 用户选择最佳UX方案，动画和云端同时进行，用户感知不到等待。

## 结果揭晓方式

| Option | Description | Selected |
|--------|-------------|----------|
| 胶囊打开揭晓（推荐） | 胶囊从中间裂开，里面是一颗珍珠+奶茶名，和真实扭蛋完全对应 | ✓ |
| 胶囊消失+卡片飞入 | 胶囊炸散/消失，奶茶卡片从侧面飞入 | |
| 跳转到结果页 | 弹出动画后直接跳转result页面 | |

**User's choice:** 胶囊打开揭晓（推荐）
**Notes:** 最忠实于真实扭蛋机体验，打开胶囊盖的瞬间就是揭晓时刻。
