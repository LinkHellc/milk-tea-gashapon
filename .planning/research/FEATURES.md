# Feature Research — 奶茶扭蛋机

**Domain:** 盲盒抽卡类小程序 / 奶茶选择工具
**Researched:** 2026-04-23
**Confidence:** MEDIUM (基于盲盒/扭蛋类小程序通用模式，非特定产品验证)

---

## Feature Landscape

### Table Stakes (用户默认期待)

缺失这些功能，产品会感觉不完整或体验断裂。

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| **微信头像昵称展示** | 用户习惯看到自己的身份标识 | LOW | 微信授权获取，展示在结果页/个人中心 |
| **抽取动画与交互** | 核心体验必须有仪式感 | MEDIUM | 胶囊弹出 + 翻开效果，3-5秒动画 |
| **结果展示与揭示** | 抽到什么必须清晰展示 | LOW | 奶茶名 + 一句话介绍 + 图片 |
| **分享功能** | 用户想炫耀"手气" | MEDIUM | 生成分享图 (canvas渲染)，含奶茶名+品牌+小程序码 |
| **历史记录** | 知道自己抽过什么、抽了几次 | LOW | 本地存储，展示最近10条 |
| **音效反馈** | 增强仪式感和沉浸感 | LOW | 扭蛋音效 + 结果揭晓音效 |
| **加载/loading态** | 网络等待时不能白屏 | LOW | skeleton屏或品牌logo动画 |
| **重新抽取** | 手气不好想再试试 | LOW | 结果页显眼按钮 |

### Differentiators (差异化竞争力)

这些是产品的竞争点，与"今天喝什么"核心价值强绑定。

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| **奶茶杯形态扭蛋机** | 契合奶茶主题，代入感强 | MEDIUM | 每个品牌独立视觉风格，杯身即扭蛋机 |
| **成就徽章系统** | 提高打开频次和留存 | MEDIUM | 隐藏款达成、集齐品牌、连续签到等 |
| **积分奖励机制** | 用户有目标感，愿意回来 | MEDIUM | 每扭获得积分，累积兑换权益 |
| **隐藏款奶茶** | 制造话题，满足收集欲 | LOW | 概率更低但更有趣的奶茶款式 |
| **品牌切换体验** | 9个品牌可以自由探索 | LOW | 底部tab或首页品牌列表 |
| **用户自定义胶囊** | 参与感，让用户"养"自己的扭蛋机 | HIGH | 用户可往当前品牌添加奶茶款型 |

### Anti-Features (看似需要，实则问题)

| Feature | Why Requested | Why Problematic | Alternative |
|---------|---------------|-----------------|-------------|
| **店铺匹配/LBS** | 帮用户找到附近门店 | 超出工具定位，且涉及商家数据 | 只做品牌选择，门店由用户自行搜索 |
| **奶茶百科详情** | 想了解更多奶茶信息 | 破坏"即用即走"定位，偏离核心 | 一句话介绍足够，深信息不符合快速决策场景 |
| **热量/糖分展示** | 健康顾虑 | 与"惊喜感"冲突，计算负担重 | 可在成就徽章中轻量提示，不做专门tab |
| **用户评价系统** | 想看其他人选什么 | 数据收集成本高，容易有水军 | 不做，用"大家都抽到过"类文案代替 |
| **频繁插播广告** | 变现需求 | 打断抽取体验，招致用户反感 | 结果页底部banner + 首页激励视频入口(用户主动触发) |
| **复杂的每日任务** | 提高留存 | 让用户感觉像上班，疲劳感 | 轻量积分，用"顺手"代替"必须" |

## Feature Dependencies

```
[微信授权登录]
    └──requires──> [头像昵称展示]

[抽取核心动画]
    └──requires──> [结果展示]
                       └──requires──> [分享生成]

[积分系统]
    └──requires──> [抽取记录]
                       └──requires──> [成就徽章计算]

[成就徽章]
    └──enhanced by──> [隐藏款奶茶] (稀有成就触发)

[生成分享图]
    └──requires──> [小程序码获取] (需微信云开发权限)
```

### Dependency Notes

- **微信授权登录 requires 头像昵称展示:** 授权后才有userid，用于积分绑定和历史记录
- **抽取核心动画 requires 结果展示:** 动画完成后必须展示奶茶结果，不能只有动画
- **积分系统 requires 抽取记录:** 每次抽记录到storage，累计积分
- **成就徽章 enhanced by 隐藏款:** 抽到隐藏款触发特殊徽章，满足收集欲

## MVP Definition

### Launch With (v1)

最小可验证产品 — 验证"扭蛋选奶茶"这个核心玩法是否成立。

- [ ] **微信授权 + 头像昵称** — 身份标识，LOW complexity，必做
- [ ] **品牌列表 (9个)** — 入口，LOW complexity，必做
- [ ] **抽取动画 (胶囊弹出+翻开)** — 核心体验，MEDIUM complexity，必做
- [ ] **结果展示 (奶茶名+一句话介绍)** — 抽到什么，LOW complexity，必做
- [ ] **重新抽取** — 手气不好再试，LOW complexity，必做
- [ ] **分享图生成** — 传播裂变，MEDIUM complexity，必做
- [ ] **音效反馈** — 仪式感增强，LOW complexity，必做
- [ ] **历史记录 (本地)** — 用户知道自己抽过，LOW complexity，必做

### Add After Validation (v1.x)

核心验证通过后，提升留存和趣味性。

- [ ] **积分系统** — 提高打开频次，trigger: 用户开始频繁使用
- [ ] **成就徽章** — 收集欲满足，trigger: 用户开始有重复使用
- [ ] **隐藏款奶茶** — 话题性，trigger: 用户开始觉得"普通奶茶没意思"

### Future Consideration (v2+)

产品站稳后扩展。

- [ ] **用户自定义胶囊** — 让用户添加自己的奶茶到品牌里，HIGH complexity，trigger: 高留存用户需要个性化
- [ ] **品牌主题扭蛋机皮肤** — 每个品牌独特视觉，MEDIUM complexity
- [ ] **社交排行榜** — 好友间比较"手气"，trigger: 分享率数据分析后决策

## Competitor Feature Analysis

微信小程序生态中，盲盒/抽卡类产品常见形态：

| Feature | 常见做法 | 奶茶扭蛋机做法 |
|---------|----------|---------------|
| 抽取动画 | 简单旋转或滑动揭示 | 胶囊弹出+翻开 (差异化: 物理感) |
| 分享图 | 纯文字或emoji | 奶茶名+品牌+小程序码 (增强品牌感) |
| 用户体系 | 无头像或仅微信授权 | 微信头像昵称 (标准化) |
| 奖励机制 | 积分或虚拟货币 | 积分+成就徽章双系统 (增强目标感) |
| 历史记录 | 云端存储 | 本地存储 (MVP阶段降低复杂度) |
| 品牌视觉 | 统一模板 | 每个品牌独立视觉 (差异化: 品牌即主题) |
